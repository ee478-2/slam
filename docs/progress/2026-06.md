## 2026-06-08 (AprilTag front/side anchor weighting)

- Added signboard-axis filtering for AprilTag global anchors (`a87230e`):
  each new anchor update is decomposed into the known signboard front/normal
  axis and side/tangent axis, using `yaw_deg` from `config/global_map.yaml`.
- Default behavior trusts the front axis fully (`front_axis_weight=1.0`) and
  damps the side axis (`side_axis_weight=0.25`). This is not a hard rejection:
  a consistent side correction still converges, just more slowly.
- `apriltag_global_localization.launch` exposes `axis_filter_enabled`,
  `front_axis_weight`, and `side_axis_weight`; `slam global-loc` can tune side
  damping with `SLAM_APRILTAG_SIDE_WEIGHT`.
- `/global_localization/selected_tag` now includes axis-filter diagnostics plus
  `raw_anchor_error_m` before filtering.

  `bash -n scripts/slam_aliases.sh`, XML parse for
  `launch/apriltag_global_localization.launch`, and scoped `git diff --check`

## 2026-06-08 (AprilTag anchor smoothing)

- Added a short-window smoothing pass for stable direct AprilTag anchors
  (`e1b717d`): after a signboard passes the consecutive-frame gate, its
  RTAB-frame observed tag pose is smoothed before solving `global_map -> map`.
- Filter choice: x/y/z use a median over the latest samples, yaw uses a
  circular mean. Default `smoothing_window_size` is 5 and can be tuned through
  `SLAM_APRILTAG_SMOOTHING_WINDOW` when running `slam global-loc`.
- Decision: no hard jump rejection was added. A real large correction can be
  valid after RTAB relocalization or map recovery; requiring consistent samples
  through the window damps one-frame noise without permanently blocking a real
  jump.
- `/global_localization/selected_tag` now reports `smoothing_window_samples`
  and `smoothing_window_size`.
- Verification: Python byte-compile for `scripts/apriltag_global_localizer.py`,

  `launch/apriltag_global_localization.launch`, and scoped `git diff --check`

## 2026-06-06 (flat-ground RTAB prior)

- Added the missing flat-ground prior for real RTAB-Map (`94976ba`):
  `rtabmap_realsense.launch` now defaults `rtabmap_args` to
  `--delete_db_on_start --Reg/Force3DoF true`.
- Rationale: the real robot moves on flat ground and this launch uses
  `frame_id=base_link`, so `Reg/Force3DoF=true` constrains registration to
  x/y/yaw and prevents RTAB visual odometry / map optimization from accumulating
  roll, pitch, or z. This is different from the old optical-frame trap: forcing
  3DoF in an optical camera frame can block real forward motion, but forcing it
  in `base_link` coordinates matches the robot kinematics.
- Updated `docs/slam_commands.md` so persistent-map runs keep the prior with
  `rtabmap_args:="--Reg/Force3DoF true"` instead of clearing all RTAB args.
- Verification: `roslaunch --nodes slam rtabmap_realsense.launch`,
  `git diff --check`, and `catkin_make --pkg slam` passed. No live robot
  `/rtabmap/odom` roll/pitch measurement was run after the change.

## 2026-06-06 (planar AprilTag anchor solve)

- Fixed a remaining AprilTag/global-map mismatch (`1fb2d38`): the global anchor
  no longer computes a full 6-DoF tag alignment and then projects it to planar
  x/y/yaw. That projection can move the tag's x/y point when the observed tag
  pose includes roll, pitch, or camera-height geometry.
- The node now solves the planar transform directly:
  `global_tag_xy = global_map_to_map_xy + R(yaw) * rtab_tag_xy`, so the
  detected RTAB tag point lands on the configured signboard point in
  `global_map.yaml`.
- `/global_localization/selected_tag` now includes `anchor_error_m`; it should
  be near zero for the selected tag. If the displayed signboard still disagrees
  after this, the next suspect is the physical tag-id-to-signboard table or the
  configured signboard coordinates, not the planar transform math.
- Verification: Python byte-compile, targeted planar-anchor numerical check,
  `git diff --check`, and `catkin_make --pkg slam` passed. No live robot
  `/tag_detections` snapshot was run.

### Non-obvious findings (the reason this works)
- **USB2 CAN run RGB-D SLAM** — but only if you cut the stream set:
